# 《南城客栈》连锁餐饮后台管理系统

本项目设计了类似于蜜雪冰城这样的加盟连锁店的后台管理系统。功能模块方面设计了包含权限管理、用户管理、门店管理、采购管理、库存管理、原料管理、菜单管理、营销管理、订单管理模块。数据库设计方面设计了对应数据库的53张数据表。使用Apifox设计了100+个功能接口。项目使用的技术栈：java、mysql、redis、springboot、mybatis、RabbitMq、webSocket

## 项目特点

1.实现了较为完整的权限管理功能。系统对后台用户的权限管理使用了“用户-部门-权限”的管理方案，大部分接口都加入了独立的权限码，通过给角色绑定权限码的方式控制角色可以访问的接口。将角色分配给用户即可实现不同的权限控制。同时为了实现数据隔离，在一些操作敏感数据的接口添加了数据范围，一般包含3种级别，分别是个人范围、部门范围、全部数据。该模块使用了Redis缓存用户权限信息，便于快速鉴权，修改用户权限后使用了先改sql后删除缓存的方式保证数据的最终一致性。

2.设计了较为完善的原料管理。原料采购的大致流程被设计为：各部门可通过提交采购申请给本部门的采购员（使用webSocket主动通知），采购员受理采购申请后寻找合适的供应商并创建采购订单，采购完成后将采购订单交给仓管员检查入库。原材料售卖的大致流程被设计为：各门店向其上级部门提交要货申请，上级部门收到付款后仓管员出库原料并创建出库单，当门店店长收到原材料后系统自动将原材料入库门店仓库。

3.设计了较为完善的菜单管理。可以为每个菜品设置标准配方，配方包含菜品的制作步骤以及每一步需要使用的原材料和数量。并且还设计了配置项管理，如甜度、温度、杯型等维度的自定义配置项。每个菜品会单独与配置值关联从而实现价格与原材料用量的精准控制。套餐方面设计了套餐内菜品自定义，管理员在后台创建菜品分组后，可为一个套餐绑定多个分组。顾客在购买套餐时即可在套餐绑定的多个分组中自由搭配菜品。

4.实现了较为丰富的营销模块。主要设计了“活动”和“优惠券”两大营销类型。活动方面设计了买送、买折、时段优惠活动。优惠券方面设计了折扣券、满减券、商品券三种类型。活动和优惠券均通过“门店”，“商品”两个维度来控制生效范围。活动一般由门店申请或管理员统一创建。而优惠券由管理员创建后可设置对应价格，一般需要顾客购买获得。

5.实现了较为完整的订单管理。涉及顾客下单全流程，包括修改购物车、创建订单、订单状态变更。订单号的生成使用了lua脚本在redis中循环生成唯一id+时间戳的方式生成唯一订单。重点实现了订单取消或退款时涉及的原料及优惠券回滚。对于超时未支付的订单使用了RabbitMq的死信交换机实现了延时关闭订单的功能，并通过定时器定时扫描来取消超时未支付的订单。
